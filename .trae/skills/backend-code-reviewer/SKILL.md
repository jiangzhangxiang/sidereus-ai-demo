# 后端代码审查技能

## 审查范围

本技能专门用于审查后端代码，重点关注：

### 技术栈规范
- **NestJS 10+**：模块化设计、依赖注入、装饰器使用
- **Fastify 适配器**：性能优化、中间件配置、路由处理
- **TypeORM**：实体定义、关系映射、查询优化
- **PostgreSQL**：数据库设计、索引优化、事务处理
- **TypeScript**：接口定义、类型安全、错误处理

### 代码质量维度
1. **架构设计**：模块化程度、分层结构、依赖关系
2. **API设计**：RESTful规范、错误处理、响应格式
3. **数据库操作**：实体关系、查询性能、事务管理
4. **安全性**：输入验证、SQL注入防护、认证授权
5. **性能优化**：查询优化、缓存策略、异步处理
6. **可测试性**：单元测试、集成测试、Mock使用

### 审查重点
- 控制器设计规范和RESTful API实现
- 服务层业务逻辑的合理性和可复用性
- 实体定义的完整性和关系映射的正确性
- DTO验证机制的严格性
- 错误处理机制的完整性
- 数据库查询的性能和安全性

## 审查流程

1. **架构检查**：验证模块化设计和分层架构
2. **API设计审查**：检查RESTful规范和响应格式
3. **数据库操作分析**：审查TypeORM使用和查询优化
4. **安全性评估**：检查输入验证、错误处理和数据安全
5. **性能评估**：识别性能瓶颈和优化机会
6. **可测试性检查**：验证代码的可测试性和测试覆盖率

## 输出格式

提供结构化的审查报告，包括：
- 主要问题列表（按严重程度排序）
- 具体改进建议
- 代码示例演示正确写法
- 相关技术文档链接
- 安全性风险评估

## 特别关注点

- **模块化设计**：app.module、feature.module的正确实现
- **依赖注入**：@Injectable装饰器的正确使用
- **装饰器使用**：@Controller、@Get、@Post等装饰器的规范使用
- **数据库连接**：ConfigService配置和环境变量管理
- **错误处理**：全局异常过滤器和自定义异常类
- **请求验证**：DTO类的class-validator装饰器使用
