# 前端代码审查技能

## 审查范围

本技能专门用于审查前端代码，重点关注：

### 技术栈规范
- **React 19 + TypeScript**：组件结构、hooks使用、Props接口定义
- **Vite 配置**：构建优化、插件配置、环境变量管理
- **Ant Design**：组件使用规范、样式定制、主题配置
- **React Router v6**：路由配置、导航守卫、懒加载实现
- **Zustand**：状态管理、store设计、异步处理

### 代码质量维度
1. **类型安全**：TypeScript类型定义是否完整、准确
2. **组件设计**：单一职责原则、可复用性、可维护性
3. **性能优化**：React.memo、useMemo、useCallback的正确使用
4. **代码风格**：ESLint规则、Prettier格式化、命名规范
5. **最佳实践**：错误边界、加载状态、可访问性

### 审查重点
- 函数组件和hooks的正确使用
- Props接口定义的完整性
- 状态管理的合理性
- 组件间的解耦
- 性能优化的必要性
- 样式组织的一致性

## 审查流程

1. **技术栈检查**：验证是否符合项目指定的技术栈规范
2. **代码结构分析**：检查文件组织、模块化程度
3. **类型定义审查**：确保TypeScript类型定义准确且完整
4. **性能评估**：识别潜在的性能问题和优化机会
5. **最佳实践验证**：符合React和TypeScript最佳实践
6. **可维护性检查**：代码的可读性、可扩展性

## 输出格式

提供结构化的审查报告，包括：
- 主要问题列表（按严重程度排序）
- 具体改进建议
- 代码示例演示正确写法
- 相关技术文档链接
